# ping-me

Push notifications to your iOS phone when a console on your computer needs you.
Free, no account, ~80 lines of bash, one config file.

Built on [ntfy](https://ntfy.sh) — open protocol, swap to a self-hosted server
later with a one-line config change.

## Install

```bash
./install.sh
```

Then on iOS:

1. Install the **ntfy** app (App Store, free).
2. Add subscription, topic = the one printed by `install.sh`, server = `https://ntfy.sh`.
3. Test: `ping-me "hello"` → push arrives on phone.

## Usage

```bash
ping-me "build finished"                  # arbitrary message
ping-me -- npm run build                  # run a command, ping on exit (with duration + status)
ping-me --hook                            # Claude Code Stop hook (reads JSON on stdin)
```

### Wire as Claude Code Stop hook

Replace the body of `~/workflow/hooks/notify-on-stop.sh` with:

```bash
#!/usr/bin/env bash
osascript -e 'display notification "Task complete" with title "Claude Code"' 2>/dev/null
"$HOME/.local/bin/ping-me" --hook 2>/dev/null
exit 0
```

You get the macOS banner *and* the iOS push.

## Config

`~/.config/ping-me/config`

```sh
NTFY_URL="https://ntfy.sh"      # swap for self-hosted later
NTFY_TOPIC="ping-me-xxxxxxx..." # long random, generated by install.sh
NTFY_TOKEN=""                   # optional, for auth on self-hosted
```

## Security note

Public ntfy.sh topics are world-readable to anyone who guesses the topic name.
`install.sh` generates a 32-char random topic — keep it secret, treat it like a
password. If it leaks, regenerate by emptying `NTFY_TOPIC` and re-running install.

For sensitive payloads, self-host ntfy with auth and flip `NTFY_URL` /
`NTFY_TOKEN`. No code change needed.
