## 1. Context

The [console model spec](./2026-05-12-console-model-design.md) defines *what*
a console is. This spec defines:

- The Mac-side **agent** that exposes consoles over the network.
- The **transport** between the phone and that agent.
- The **auth** model.
- The **API** the phone will speak.
- How `ping-me` feeds the `waiting_for_input` signal back into the agent.

Mobile UI is still out of scope.

## 2. Out of scope

- Mobile UI design and code.
- Cross-platform agent (Linux/Windows) вҖ” see roadmap.
- Android client вҖ” see roadmap.
- Multi-Mac federation.
- A Cloudflare-Tunnel transport вҖ” see roadmap; not in v1.

One static Go binary. One launchd plist. Two listeners: localhost and the
machine's tailnet IP.

## 4. Language & runtime

**Go**, latest stable. Reasons:

- Static binary вҶ’ zero install friction, no venv, no Homebrew Python drift.
- Stdlib covers HTTP, WebSocket (via `golang.org/x/net/websocket` or
  `nhooyr.io/websocket`), JSON, file watching (`fsnotify`).
- Idiomatic for small daemons; easy to cross-compile to Linux later.

## 5. Process model

Run as a **launchd user-agent**:

- Plist path: `~/Library/LaunchAgents/com.valentindumas.consoles-hub.plist`
- `RunAtLoad: true`, `KeepAlive: { SuccessfulExit: false }`
- Logs: `~/Library/Logs/consoles-hub/agent.log` + `agent.err`
- Binary path: `~/.local/bin/consoles-hub-agent` (symlinked from the repo build)

`launchctl bootstrap gui/$UID вҖҰ` to install; `launchctl kickstart -k` to restart.

## 6. Transport

**Tailscale + HTTP/WebSocket.** The phone joins your tailnet; the agent binds
to the Mac's tailnet IP. No port-forwarding, no public exposure, WireGuard
encryption end-to-end.

Binding rule:

```
agent listens on:
  - 127.0.0.1:7820         (always вҖ” local clients, curl)
  - <tailnet-ip>:7820      (resolved at startup via tailscale ip -4)
```

If the tailnet IP is unavailable (Tailscale not running), the agent logs a
warning and serves localhost only. It does **not** fall back to `0.0.0.0`.

TLS is **not** terminated by the agent. The tailnet already encrypts in
transit. If we ever expose beyond Tailscale, a reverse proxy adds TLS вҖ” agent
stays plain HTTP.

## 7. Auth

Two layers:

1. **Network** вҖ” Tailscale ACLs restrict which tailnet devices can reach the
   Mac on port 7820. Default ACL: only the user's own devices.
2. **Bearer token** вҖ” every HTTP and WS request must carry
   `Authorization: Bearer <token>`. Token is generated by `install.sh`,
   stored in `~/.config/consoles-hub/agent.toml`, and surfaced once on
   install for the mobile client.

The token is the only defense if Tailscale ACLs are misconfigured. It is also
the only thing distinguishing a legitimate phone from another device the user
owns. Keep it long (вүҘ32 chars), random, rotatable via config.

Loopback requests (`127.0.0.1`) bypass the token check вҖ” debugging with curl
should be friction-free.

## 8. REST API

Base URL: `http://<tailnet-ip>:7820`

| Method | Path | Body | Returns |
|---|---|---|---|
| `GET` | `/healthz` | вҖ” | `{ "ok": true, "tmux": true }` |
| `GET` | `/consoles` | вҖ” | `[ ConsoleRecord ]` (see console-model spec В§5) |
| `GET` | `/consoles/:id` | вҖ” | `ConsoleRecord` |
| `GET` | `/consoles/:id/buffer?scrollback=N` | вҖ” | `{ "text": "...", "captured_at": "..." }` |
| `POST` | `/consoles/:id/send` | `{ "text": "...", "enter": true }` | `{ "ok": true }` |

Error shape:

```json
{ "error": "pane_not_found", "message": "no pane with id %23" }
```

Common errors: `pane_not_found`, `tmux_unavailable`, `unauthorized`, `invalid_input`.

`scrollback=N` is capped at 5000 lines.

## 9. WebSocket вҖ” live pane stream

`GET ws://<tailnet-ip>:7820/consoles/:id/stream` (upgrades to WS).

Server pushes JSON messages. Two message types:

```json
{ "type": "snapshot", "text": "...", "captured_at": "..." }
{ "type": "delta",    "text": "...", "captured_at": "..." }
```

- One `snapshot` is sent immediately on connect (full visible buffer).
- Subsequent `delta` messages contain the new tail since the last message.

Cadence: at most one message per 100 ms (coalesce). Server detects no-change
by hashing the last capture; idle panes emit nothing.

The client can also send messages over the same socket to **type into the
pane**:

```json
{ "type": "send", "text": "y", "enter": true }
```

This is equivalent to `POST /consoles/:id/send` but on the existing socket
(lower latency, no extra auth round-trip вҖ” token is checked at WS upgrade).

Closing the socket has no side effect on the pane.

## 10. `ping-me` вҶ” agent signal channel

`ping-me --hook` must tell the agent **which pane** invoked it so the agent
can flip `waiting_for_input` on that record.

Contract:

- `ping-me --hook` reads `$TMUX_PANE` (tmux sets this in every shell child).
- It appends one NDJSON line to `~/.local/state/consoles-hub/signals.ndjson`:
  ```json
  {"pane_id":"%23","at":"2026-05-12T10:21:03Z","kind":"waiting"}
  ```
- The agent watches that file via `fsnotify` and updates in-memory state.

If `$TMUX_PANE` is unset (hook ran outside tmux), ping-me still sends the iOS
push but writes `{"pane_id":null,...}`. Agent ignores `null` signals.

The signal file is append-only; the agent never rewrites it. A daily rotate
(keep 7 days) is the agent's responsibility вҖ” out of scope for `ping-me`.

## 11. Logging & health

- All logs are line-oriented JSON to `~/Library/Logs/consoles-hub/agent.log`.
- `GET /healthz` reports liveness + whether `tmux` is reachable.
- On startup, agent logs: bind addresses, tmux version, signal file path,
  config path, and the (masked) token fingerprint.

## 12. Config & install

```
~/.config/consoles-hub/agent.toml
```

```toml
listen_port = 7820
bind_tailnet = true            # if false, localhost only
token        = "вҖҰ32+ charsвҖҰ"   # generated by install.sh
signal_file  = "~/.local/state/consoles-hub/signals.ndjson"
buffer_scrollback_cap = 5000
```

`install.sh` (in the agent's directory) will:

1. Build the binary (`go build`).
2. Install to `~/.local/bin/consoles-hub-agent`.
3. Generate config + token if missing.
4. Write the launchd plist and bootstrap it.
5. Print the tailnet URL + token for the iOS client.

Existing `ping-me/install.sh` is unaffected.

## 13. Roadmap (explicitly future, not v1)

Documented so we don't paint ourselves into a corner:

- **Cloudflare Tunnel as alternate transport.** Same agent, same API. The
  tunnel just exposes the localhost port through Cloudflare's edge instead
  of the tailnet. Useful when Tailscale isn't an option.
- **Cross-platform agent.** The agent's only OS-specific surface is launchd.
  Replace with systemd unit on Linux and a Service Control Manager wrapper
  on Windows. The tmux dependency holds on Linux/macOS; on Windows we'd
  swap to Windows Terminal sessions or WSL.
- **Android client.** REST + WS API is language-agnostic; an Android client
  consumes the same endpoints. The bearer token + tailnet model carries over.

These items must not be implemented until the user explicitly asks.

## 14. Open questions

- Whether to expose `cmd` aliasing (`node` вҶ’ `claude`) here or push it to the
  mobile UI. Leaning UI; keep the agent honest.
- Buffer hash algorithm for delta detection вҖ” pick at implementation time
  (xxhash or just `sha256` of the captured text, both fine).
