## Spacing Scale

Pico.css classless does not expose custom CSS spacing tokens. This project overrides with inline `style` or the existing `app.css` margin/padding values. All values are multiples of 4 px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, `padding-left: 4px` on sidebar active link |
| sm | 8px | Compact element spacing, row-action gaps |
| md | 16px | Default element padding — e.g. `.banniere-success { padding: 16px }` |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Upload `<progress>` element: full width of its containing `<form>` column (no fixed width token)
- Touch targets: minimum 44x44px for all `<button>` and `<a role="button">` (WCAG 2.5.5)

Source: `app.css` observed values + 8 px scale convention confirmed D-42 Phase 1.

## Interaction Contracts

### Upload flow (D-116)

1. User navigates to `/coffre/upload` or uses inline form on `/travaux/:id` PJ section.
2. `<form enctype="multipart/form-data" method="POST">` — single file `<input type="file" accept=".pdf,.jpg,.jpeg,.png,.heic,.webp">`.
3. Label reads: "Fichier — PDF, JPG, PNG, HEIC, WebP — max 50 Mo"
4. On submit: server validates magic-bytes + MIME, converts HEIC → JPEG, persists, redirects.
5. `<progress>` native element shown during form submission (CSS `form[data-uploading] progress { display: block }`). Minimal JS: set `data-uploading` on form `submit` event.
6. On success: redirect to `/justificatifs/:id` with `banniereSuccess="Document ajouté."`.
7. On error (413, bad MIME, invariant): re-render form with `aria-invalid="true"` on the file input + `.error-msg` below.

### File view (D-117)

- PDF / WebP / JPEG (converted HEIC): `<a href="/justificatifs/:id/fichier" target="_blank" rel="noopener">Ouvrir le fichier</a>` — opens in browser native viewer.
- JPG / PNG (original): inline `<img src="/justificatifs/:id/fichier" alt="{titre}">` in the detail section on `/justificatifs/:id`. Constrained to `max-width: 100%; height: auto;`.

### Soft-delete flow (D-109)

1. "Mettre en corbeille" button on `/coffre` row actions (`.row-actions` — visible on hover/focus).
2. Click opens `confirm-dialog.ejs` with message: "Ce document sera déplacé vers la corbeille. Vous pourrez le restaurer ou le purger définitivement depuis /coffre/corbeille."
3. Confirm POST → redirect to `/coffre` with `banniereSuccess="Document déplacé vers la corbeille."`.

### Purge flow (D-109)

1. "Purger définitivement" button on `/coffre/corbeille` — only rendered if `peutEtrePurge() === true`.
2. If `peutEtrePurge() === false`: no button. Instead, show `.banniere-warning` inline in the row: "Conservation légale obligatoire jusqu'au {date}."
3. If enabled: `confirm-dialog.ejs` with message: "Cette action est irréversible. Le fichier sera supprimé définitivement."

### Ticket statut transition (D-114)

- Statut badge: pill-style `<span>` with inline color: `ouvert` → `#1d4ed8` text on light blue bg, `en_cours` → `#d97706` text on amber bg, `clos` → `#16a34a` text on green bg, `annule` → `#6b7280` text on gray bg.
- Statut transitions are manual only — no auto-transition V1. "Clore" button on fiche détail opens an inline `<form>` section (not a dialog) to capture `dateCloture` + `coutReelTtc`.

### Filter behavior on `/coffre` (D-110)

- Filter form: `<form method="GET" action="/coffre">` — all filters submit via GET. URL reflects filter state (bookmarkable).
- Search field: `<input type="search" name="q" placeholder="Titre, notes, nom de fichier…">`.
- Dropdowns: `<select>` for Bien, Locataire, Année, Type — each with an "Tous" default option (value="").
- No live filtering V1 — full page reload on submit.

## Accessibility Contract

All Phase 4 pages must satisfy WCAG 2.1 AA. Specific obligations:

| Element | Requirement |
|---------|------------|
| `<input type="file">` | Explicit `<label>` with id reference. No `placeholder` alone. |
| Upload format hint | Associated via `aria-describedby` pointing to the hint paragraph. |
| `<progress>` | Include `aria-label="Upload en cours"` and `aria-live="polite"` on containing wrapper. |
| Upload success | `banniere-success.ejs` uses `role="status" aria-live="polite"` — already correct. |
| Rétention block | Use `.banniere-warning` (existing) — `aria-live` not required (non-critical). |
| Purge block message | `role="status"` on warning row cell. |
| Filter form | Each `<select>` has explicit `<label>`. |
| Coffre table | `aria-label="Justificatifs"`. Sortable columns not required V1 (static sort). |
| Ticket table | `aria-label="Tickets travaux"`. |
| Statut badge | Use `<span>` with visible text — no color-only encoding. |
| Inline image preview | `<img alt="{titre du Justificatif}">` — title is always present (required field). |
| `<a target="_blank">` | Always include `rel="noopener"` + screen-reader hint via `.sr-only` "(ouvre dans un nouvel onglet)" |
| Confirm dialogs | Reuse `confirm-dialog.ejs` which sets `autofocus` on cancel button — correct focus trap. |
| Keyboard navigation | All row actions visible on `tr:focus-within` — already in `app.css`. |
| `prefers-reduced-motion` | Upload progress animation: none (native `<progress>` has no custom animation). Auto-dismiss `.banniere-success` respects existing `@media (prefers-reduced-motion: reduce)` in `app.css`. |
| Contrast | `#dc2626` on white = 4.63:1 PASS. `#1d4ed8` on white = 5.9:1 PASS. `#d97706` on white = 3.13:1 — use on colored bg only (amber bg `#fef3c7` gives sufficient contrast for text label). |
