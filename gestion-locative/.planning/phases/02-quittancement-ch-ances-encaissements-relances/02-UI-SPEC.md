# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2 : Quittancement — Échéances, Encaissements, Relances.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | D-20 (Phase 1 locked) : Pico.css classless — no component framework |
| Preset | not applicable | No components.json — semantic HTML + Pico.css |
| Component library | none — semantic HTML + Pico.css | D-20 |
| Icon library | none V1 — Unicode symbols ou inline SVG aria-hidden | Phase 1 default |
| Font | system-ui (Pico.css default stack) | D-20 default |

**Rationale :** Pico.css classless : les éléments HTML sémantiques sont stylés nativement. Les partials EJS constituent le modèle de composants (D-11). Pas de Tailwind, pas de shadcn, pas de Radix. Tout nouveau composant Phase 2 suit le même pattern : balise sémantique + classe CSS custom minimale dans `app.css` si Pico.css ne couvre pas le besoin.

## Spacing Scale

Identique Phase 1. Valeurs multiples de 4, aucune valeur arbitraire. Source : D-46.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Écart icône-texte, padding inline badge/pill statut |
| sm | 8px | Padding cellule tableau, espacement compact |
| md | 16px | Écart champs formulaire, padding carte, espacement nav |
| lg | 24px | Padding de section, espacement fieldset |
| xl | 32px | Écart colonnes layout, séparation majeure formulaire |
| 2xl | 48px | Padding vertical haut/bas de page |
| 3xl | 64px | Offset vertical maximal conteneur |

Exceptions : cibles de clic/tap minimum 44px de hauteur (WCAG 2.5.5 AAA — via `padding` uniquement, pas de token override). S'applique en particulier aux boutons d'action de relance et aux liens de téléchargement PDF.

## Typography

Identique Phase 1. System-ui, pas de chargement de police externe (local-first : fonctionne hors-ligne). Source : D-48.

| Role | Size | Weight | Line Height | Max Line Length |
|------|------|--------|-------------|-----------------|
| Body | 16px | 400 (regular) | 1.5 | 72 chars |
| Label | 14px | 600 (semibold) | 1.4 | — |
| Heading | 20px | 600 (semibold) | 1.2 | 60 chars |
| Display | 28px | 600 (semibold) | 1.2 | 50 chars |

Règles :
- Maximum 3 tailles distinctes par vue (heading + body + label — Display réservé à l'éventuelle page d'activation de bail uniquement).
- Maximum 2 graisses sur toute vue : 400 regular + 600 semibold.
- Corps jamais en dessous de 16px. Pas de tout-majuscule dans les phrases.
- Montants monétaires dans les tableaux : 16px, 400, alignés à droite (`text-align: right`).
- Numéro de quittance (ex. `2026-001`) : 16px, 600, `font-variant-numeric: tabular-nums` pour alignement colonne.

## Color

Palette Pico.css avec surcharges sémantiques. Toutes les couleurs respectent WCAG 2.1 AA (4.5:1 corps, 3:1 grand texte). La couleur n'est jamais le seul différenciateur — toujours accompagnée d'une icône Unicode ou d'un libellé texte. Source : D-44, D-47.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (clair) / `#13171f` (sombre — défaut Pico) | Fond de page, surfaces |
| Secondary (30%) | `#f3f4f6` (clair) / `#1e2430` (sombre) | Sidebar nav, zebra table, fond cards |
| Accent (10%) | `#1d4ed8` (Pico primary blue) | Voir liste réservée ci-dessous |
| Destructive | `#dc2626` (Pico red-600 équivalent) | Actions destructives uniquement |
| Warning | `#d97706` (Pico amber) | Warning sur-paiement, badge `partiellement_payee`, warning rétroactif > 2 ans, warning quittance invalidée |
| Success | `#16a34a` (Pico green) | Bannière succès persistée, badge `payee` |
| Retard | `#dc2626` — même que Destructive | Badge `en_retard` (dérivé, pas un statut stocké) — toujours accompagné du texte "En retard" |
| Neutre | `#6b7280` (Pico grey) | Badge `en_attente`, texte atténué (ligne `annulee`) |

**Accent réservé exclusivement à (liste exhaustive) :**
1. Bouton d'action primaire ("Enregistrer", "Saisir l'encaissement", "Activer le bail") — fond plein.
2. Lien "Télécharger" avis d'échéance PDF et lien "Télécharger" quittance PDF — couleur de lien.
3. Bordure gauche de l'item de navigation actif dans la sidebar.
4. Indicateur de focus sur tout élément focusable (complément du défaut navigateur).
5. Bouton "Générer la quittance" visible uniquement quand `EcheanceLoyer.statut = 'payee'`.

Tout le reste (actions secondaires, hover ligne tableau, champs au repos) utilise la surface secondaire ou le style Pico par défaut.

### Couleurs de statut `EcheanceLoyer`

| Statut | Couleur texte/fond | Symbole Unicode | Règle d'affichage |
|--------|-------------------|-----------------|-------------------|
| `payee` | `#16a34a` vert | ✓ (U+2713) | Badge + texte "Payée" |
| `partiellement_payee` | `#d97706` amber | ◐ (U+25D0) | Badge + texte "Partiel" |
| `en_retard` (dérivé) | `#dc2626` rouge | ✕ (U+2715) | Badge + texte "En retard" (affiché si `statut != 'payee' && jour_echeance_attendue < today`) |
| `en_attente` | `#6b7280` gris | ○ (U+25CB) | Badge + texte "En attente" |

Implémentation technique : classe CSS `statut-badge` + modificateur (`--payee`, `--partiel`, `--retard`, `--attente`, `--annulee`) dans `app.css`. Jamais de couleur seule — symbole Unicode + texte systématiques.

## Component Patterns (nouveaux — Phase 2)

Les 8 partials Phase 1 sont réutilisés sans modification. Les nouveaux composants ci-dessous complètent l'inventaire.

### Badge statut `EcheanceLoyer` (`partials/statut-badge.ejs`)

```html
<span class="statut-badge statut-badge--<%= statut %>">
  <span aria-hidden="true"><%= symbole %></span>
  <span><%= libelle %></span>
</span>
```

Variables `locals` : `{ statut: 'payee'|'partiellement_payee'|'en_retard'|'en_attente'|'annulee', libelle: string, symbole: string }`.

CSS dans `app.css` :
```css
.statut-badge { display: inline-flex; align-items: center; gap: 4px; padding: 2px 8px; border-radius: 4px; font-size: 14px; font-weight: 600; }
.statut-badge--payee { background: #dcfce7; color: #16a34a; }
.statut-badge--partiellement_payee { background: #fef3c7; color: #d97706; }
.statut-badge--en_retard { background: #fee2e2; color: #dc2626; }
.statut-badge--en_attente { background: #f3f4f6; color: #6b7280; }
.statut-badge--annulee { background: #f3f4f6; color: #6b7280; text-decoration: line-through; }
```

Contraste vérifié : fond clair `#dcfce7` / texte `#16a34a` → ratio ≥ 4.5:1 WCAG AA.

### Zone warning aria-live (`partials/warning-live.ejs`)

Pour le sur-paiement (D-59) et la quittance invalidée (D-65) :

```html
<% if (locals.warning) { %>
  <aside role="status" aria-live="polite" class="warning-zone">
    <span aria-hidden="true">⚠</span> <%= locals.warning %>
  </aside>
<% } %>
```

CSS :
```css
.warning-zone { background: #fef3c7; border-left: 4px solid #d97706; padding: 8px 16px; margin: 16px 0; color: #92400e; }
```

Non-bloquant : l'utilisateur peut continuer à agir après lecture. `aria-live="polite"` — pas de `role="alert"` (le warning n'est pas une erreur bloquante).

### Bouton conditionnel relance (`partials/relance-action.ejs`)

Rendu uniquement si le niveau N est disponible selon D-71 :

```html
<% if (locals.niveauDisponible) { %>
  <button
    type="submit"
    form="form-relance-<%= echeanceId %>"
    class="relance-btn relance-btn--niveau<%= locals.niveauDisponible %>"
    aria-describedby="relance-hint-<%= echeanceId %>"
  >
    Lancer relance niveau <%= locals.niveauDisponible %>
    (<%= locals.niveauLabel %>)
  </button>
  <span id="relance-hint-<%= echeanceId %>" class="sr-only">
    <%= locals.niveauHint %>
  </span>
<% } %>
```

`niveauLabel` : `"amiable"` (1), `"ferme"` (2), `"mise en demeure"` (3). `niveauHint` : description de l'action côté screen reader ("Ouvre un brouillon email pour relance amiable").

Niveau 3 : le bouton déclenche un téléchargement PDF, pas un mailto. Le libellé devient "Télécharger la mise en demeure PDF".

### Fiche échéance — zone actions conditionnelles

La zone d'actions en bas de fiche `GET /echeances/:id` varie selon le statut. Rendu conditionnel (pas de JS — server-rendered selon `statut`) :

| Statut | Actions affichées |
|--------|------------------|
| `en_attente` | "Télécharger avis d'échéance" + "Saisir un encaissement" |
| `partiellement_payee` | "Saisir un encaissement" (complément) + bouton relance si seuil atteint |
| `payee` | "Générer la quittance" (accent, visible uniquement ici) + bouton quittance déjà émise si existante |
| `en_retard` | "Saisir un encaissement" + bouton relance niveau N |
| `annulee` | Aucune action — message "Cette échéance a été annulée." |

### Modal activation bail (D-73 + D-74)

Réutilise le partial `confirm-dialog.ejs` Phase 1 avec les propriétés suivantes.

**Modal modification bail actif (D-73) :**
- `message` : "X échéances futures non payées seront supprimées et régénérées avec le nouveau montant. Y échéances déjà encaissées resteront inchangées. Confirmer ?"
- `confirmLabel` : "Régénérer les échéances"
- `cancelLabel` : "Annuler"
- Focus par défaut : "Annuler" (safer default — D-50)

**Modal suppression bail avec activité (D-74) :**
- `message` : "Ce bail a {n_ech} échéances, {n_enc} encaissements et {n_qui} quittances. Supprimer détruirait l'historique opposable."
- `confirmLabel` : n/a — le bouton "Supprimer" est désactivé (`disabled`) avec `aria-disabled="true"` et `title="Suppression impossible : ce bail a des données liées"`
- CTA alternatif affiché : "Désactiver le bail" (bouton secondary — retire `actif_depuis`, conserve tout l'historique)

### Indicateur warning rétroactif (D-72)

Affiché sur le formulaire d'activation si `actif_depuis < today - 2 ans` :

```html
<aside role="status" aria-live="polite" class="warning-zone">
  <span aria-hidden="true">⚠</span>
  Activation > 2 ans en arrière. Vérifie les exercices fiscaux concernés
  (les échéances générées couvriront des années déjà déclarées ou à déclarer).
</aside>
```

Rendu server-side après soumission du champ `actif_depuis`. Non-bloquant.

## PDF Layout Contracts (pdfmake)

Toutes les dimensions sont en points pdfmake (1pt ≈ 0.353mm). Format A4 : 595 × 842pt. Marges : 56pt (≈ 2 cm) sur tous les côtés.

### Avis d'échéance PDF (ENC-02 — on-the-fly, D-66)

Structure JSON pdfmake :

```
[ENTÊTE — 2 colonnes]
  Colonne gauche : "AVIS D'ÉCHÉANCE DE LOYER"  (fontSize: 14, bold: true)
                   Période : "Mai 2026"          (fontSize: 12)
  Colonne droite : Date de génération            (fontSize: 10, alignment: 'right')

[SÉPARATEUR — ligne horizontale]

[BAILLEUR — LOCATAIRE — 2 colonnes]
  Gauche :  "Bailleur"  (bold) + nom_complet + adresse (3 lignes)
  Droite :  "Locataire" (bold) + nom + prenom + adresse du bien loué

[TABLEAU MONTANTS]
  | Désignation            | Montant    |
  | Loyer hors charges     | 800,50 €   |
  | Charges (forfait/prov) | 50,00 €    |
  | Total à régler         | 850,50 €   |   (bold, fontSize: 12)

[PIED DE PAGE]
  "À régler avant le {jour_echeance_attendue} du mois."
  [Si IBAN renseigné] "Virement : IBAN {iban}"
  "Document non opposable — à titre indicatif."
```

Styles pdfmake :
- `defaultStyle : { font: 'Helvetica', fontSize: 10, lineHeight: 1.4 }`
- En-têtes tableau : `fillColor: '#f3f4f6'`, `bold: true`
- Colonne montants : `alignment: 'right'`
- Total : `bold: true, fontSize: 12`

### Quittance PDF (ENC-01 — persistée, D-63)

Mentions obligatoires loi 89 art. 21 — toutes présentes et dans l'ordre légal :

```
[ENTÊTE]
  "QUITTANCE DE LOYER N° 2026-001"   (fontSize: 16, bold, centered)
  Période : "du 1er mai 2026 au 31 mai 2026"  (fontSize: 12, centered)

[BAILLEUR — LOCATAIRE — 2 colonnes]
  Gauche : "Le bailleur" (bold) + Nom Complet + Adresse complète
  Droite : "Au locataire" (bold) + Prénom Nom + Adresse du bien loué

[TABLEAU VENTILATION]
  | Désignation                    | Montant    |
  | Loyer principal hors charges   | 800,50 €   |
  | Charges locatives (forfait)    | 50,00 €    |
  | Total encaissé                 | 850,50 €   |   (bold)

[MENTION LÉGALE]
  "Le bailleur déclare avoir reçu de [Prénom Nom] la somme de [850,50 €] au titre
   du loyer et des charges pour la période du [1er mai 2026] au [31 mai 2026].
   Tous comptes apurés."   (fontSize: 10, italics: true)

[SIGNATURE]
  Lieu et date : "Fait le {date_emission}"
  "Signature du bailleur :"   [espace blanc 40pt]

[PIED DE PAGE]
  "Document établi conformément à l'article 21 de la loi n° 89-462 du 6 juillet 1989."
  "Quittance n° {numero} — non cessible — à conserver."
```

Règle d'immutabilité : le PDF est généré une seule fois, jamais ré-écrit (D-63). Si invalidée (D-65), un nouveau PDF avec numéro suivant est émis.

### Encoding mailto (niveaux 1 et 2)

L'URI `mailto:` est construit côté serveur avant rendu de la page :
- Sujet : `encodeURIComponent(sujet)`
- Corps : sauts de ligne → `%0D%0A`, accents UTF-8 → `encodeURIComponent`
- Longueur maximale de l'URI : 2 000 caractères (limite pratique Windows Mail + Outlook). Si dépassé, tronquer le corps avec mention "... [voir pièce jointe]".
- Rendu HTML : `<a href="mailto:{to}?subject={sujet}&body={corps}">Envoyer la relance amiable</a>` — lien `<a>` (pas `<button>`) car navigation externe.

## Interaction & State Contracts

### Validation formulaire encaissement

Même flow que Phase 1 (D-49) :
1. Focus sur un champ requis → pas de validation.
2. Blur sur champ requis vide → erreur inline révélée.
3. Correction du champ → erreur masquée immédiatement.
4. Submit activé dès que tous les champs requis sont valides.
5. POST failure → formulaire re-rendu avec valeurs pré-remplies + erreurs serveur. Aucune valeur effacée.
6. POST success → redirect vers fiche échéance + bannière succès (`aria-live="polite"`).

Champs requis formulaire encaissement : `date` (type="date"), `montant` (type="number", min="0.01"), `mode` (select), `echeance_id` (hidden ou select selon contexte).

### Génération quittance — flux à deux étapes

1. Utilisateur sur fiche échéance statut `payee` clique "Générer la quittance".
2. Vérification serveur : profil bailleur présent ? `EcheanceLoyer.statut = 'payee'` ? Pas de quittance déjà émise non annulée ?
3. Si OK : génération PDF + persistance fichier + `Quittance` en DB + redirect vers fiche quittance + bannière "Quittance n° 2026-001 générée avec succès."
4. Si KO (profil absent) : redirect vers `/bailleur` avec `banniereWarning` "Renseignez votre profil avant d'émettre une quittance."

### Suggestion relance — rendu conditionnel

Le serveur calcule côté route `GET /echeances/:id` si le bouton de relance est disponible :
- `niveauDisponible = null` si aucun seuil atteint → bouton non rendu.
- `niveauDisponible = 1, 2 ou 3` → bouton relance rendu avec `niveauLabel` et URI mailto ou download.
- Un clic "Lancer la relance" → POST `/relances` → création `Relance` en DB (snapshot contenu, `envoyee_le = today`) → redirect vers même fiche avec bannière "Relance niveau N enregistrée."

Pour le niveau 3 : le lien "Télécharger la mise en demeure PDF" génère le PDF on-the-fly (même pattern que l'avis d'échéance) et déclenche un téléchargement navigateur (`Content-Disposition: attachment`).

### Téléchargement PDF — comportement navigateur

Tous les PDFs (avis, quittance, mise en demeure) sont servis via une route GET dédiée :
- Avis d'échéance : `GET /echeances/:id/avis-pdf` → `Content-Disposition: attachment; filename="avis-loyer-{periode}-bail-{id-court}.pdf"`
- Quittance : `GET /quittances/:id/pdf` → `Content-Disposition: attachment; filename="quittance-{numero}-{periode}.pdf"`
- Mise en demeure : `GET /echeances/:id/mise-en-demeure-pdf` → `Content-Disposition: attachment; filename="mise-en-demeure-{periode}.pdf"`

### Bannière succès (pattern Phase 1 réutilisé)

```
"Encaissement enregistré avec succès."
"Quittance n° {numero} générée avec succès."
"Relance niveau N enregistrée."
"Profil bailleur enregistré."
"Bail activé. {N} échéances générées."
```

`aria-live="polite"`, `role="status"`, auto-disparition 5s (ou persisté si critère Phase 1).

## Accessibilité — contrat Phase 2

Reprend intégralement le contrat Phase 1 (D-44) et l'étend aux nouveaux composants.

| Vérification | Exigence Phase 2 |
|-------------|-----------------|
| Badges statut | Couleur + symbole Unicode + texte — jamais couleur seule |
| Zone warning aria-live | `role="status"` + `aria-live="polite"` — pas `role="alert"` (non-bloquant) |
| Bouton relance conditionnel | `aria-describedby` pointant vers hint sr-only décrivant l'action |
| Liens mailto | `<a>` sémantique (navigation), pas `<button>` |
| Téléchargement PDF | `<a download>` ou lien avec indication de format : "Télécharger (PDF)" |
| Tables échéances / encaissements / quittances | `<th scope="col">` + `<caption>` ou `aria-label`, sticky header, zebra |
| Modal activation bail | `<dialog>` natif + `autofocus` sur "Annuler" + trap focus + `Escape` ferme |
| Ligne `annulee` tableau | `aria-label` sur cellule statut : "Statut : Annulée" (complément au barré CSS) |
| Champ date `actif_depuis` | `<input type="date">` + `<label for>` + `aria-describedby` hint format |
| Select mode encaissement | `<label for>` + options libellées en français complet |
| Numéro quittance tableau | `font-variant-numeric: tabular-nums` — alignement colonne sans class trick |
| Contenu PDF | Hors scope WCAG (document téléchargeable) — mentionner dans le PDF : "Document généré par Gestion locative." |

Montants monétaires : tester que les assistants vocaux lisent "850 euros 50" et non "850 virgule 50 euro" — U+00A0 entre chiffre et € (Intl.NumberFormat fr-FR, Phase 1 LEARNING).
