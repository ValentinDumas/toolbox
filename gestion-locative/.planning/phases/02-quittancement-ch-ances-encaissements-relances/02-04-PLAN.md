  - constructor `(baseDir: string)` — baseDir résolu via `path.join(os.homedir(), 'Library/Application Support/gestion-locative/documents')` ou env var `GESTION_LOCATIVE_DATA_DIR`.
  - méthode `ecrireQuittance(annee: number, nomFichier: string, buffer: Buffer): Promise<string>` retourne le chemin RELATIF à baseDir (ex. `quittances/2026/quittance-2026-001-mai-2026-dupont.pdf`). Crée le dossier si absent (mkdir recursive).
  - méthode `lireQuittance(cheminRelatif: string): Promise<Buffer>` lit le fichier ; throw `FichierIntrouvable` si absent.
  - Slug locataire : `slugify(nom + '-' + prenom)` = `[a-z0-9-]` only (whitelist) — sanitisation contre path traversal.
- `genererQuittance(commande: { echeanceId }, repos+pdfRenderer+stockage+clock)` :
  - Lookup echeance (sinon EcheanceLoyerIntrouvable). Si echeance.statut !== 'payee' → throw `EcheanceLoyerNonPayee('Cette période n\'est pas entièrement réglée.')`.
  - Vérifier qu'aucune Quittance ACTIVE n'existe pour cette echeance (sinon `QuittanceDejaEmise`).
  - Lookup bailleur (sinon `BailleurAbsent('Le profil bailleur est incomplet')`).
  - Lookup bail + locataire + bien (pour PDF).
  - Transaction Kysely atomique :
    - `const annee = echeance.periodeDebut.year`.
    - `const numero = await quittanceRepo.prochainNumero(annee, trx)` — incrémente compteur dans meta et retourne nouveau numéro (UPSERT meta).
    - Quittance.creer({...}) — numero, cheminFichierRelatif (calculé d'avance).
    - quittanceRepo.enregistrer(quittance) (dans la transaction).
  - HORS transaction : générer PDF buffer puis stocker fichier sur disque. Si l'écriture échoue → le compteur a déjà été incrémenté et la Quittance enregistrée : compromis acceptable car le numéro reste valide (on peut retenter la génération PDF plus tard via une route maintenance — ou plus simple : si stockage échoue, supprimer la Quittance et réinitialiser le compteur. DÉCISION PLANNER : à l'erreur de stockage, lancer une exception et logguer ; l'utilisateur retentera. La Quittance reste en DB avec son numéro mais le PDF manquera — afficher un warning UI "PDF manquant pour la quittance 2026-XXX, régénérer depuis la fiche". Cette gestion d'erreur peut être améliorée plus tard (Phase 3+).).
  - Retourner `{ quittanceId, numero, cheminFichierRelatif }`.
- `annulerQuittance(commande, repos, clock)` : Quittance.annuler + persist (D-65). NE PAS supprimer le fichier PDF (preservation audit).
- `formatNumeroQuittance(annee: number, sequence: number): string` → `${annee}-${String(sequence).padStart(3, '0')}` (ex. (2026, 42) → "2026-042").
