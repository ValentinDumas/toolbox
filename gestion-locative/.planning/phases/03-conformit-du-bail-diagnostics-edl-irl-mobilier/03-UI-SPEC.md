|-------|-------|-------|

| sm | 8px | Compact spacing between form fields, row padding |
| md | 16px | Default element spacing, section padding inside cards |

Exceptions:
- Wizard IRL step indicator: 32px horizontal gap between step nodes (Fitts's Law — distinct tap targets).

## Typography

Pico.css provides the base typographic scale via CSS custom properties. Phase 3 inherits exactly what Phases 1-2 established (D-45, D-48).

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|-----------------|
| Body | 16px | 400 (regular) | 1.5 | Paragraphs, `<dl>` values, table cells, hint text, warning messages, the D-95 "Ne pas indexer" explanatory paragraph |
| Label | 14px | 400 (regular) | 1.4 | `<label>` on form inputs, `<small>` hints, breadcrumb items, table column headers `<th>` |
| Heading | 20px | 600 (semibold) | 1.2 | `<h2>` section headings (Diagnostics, État des lieux, Révision IRL, Historique des indexations, Checklist mobilier) |
| Display | 28px | 600 (semibold) | 1.2 | `<h1>` page title only — one per page, never repeated |

Constraints (D-45, D-48):
- Maximum 3 font sizes per view — respected (14/16/20 on detail pages; 16/20/28 on wizard steps).
- Maximum 2 font weights — respected (400 + 600).
- Body text ≥ 16px — respected.
- Line length 60–80 characters — respected via `.container` max-width (Pico.css default ~75ch).
- No all-caps in sentences.

### LOC-03 — Formulaire EDL d'entrée (`/baux/:id/edl/entree/nouveau`)

**States:**
1. **Formulaire vide** — 12 items décret 2015-981 en checklist (cases `<input type="checkbox">`) + champs `etat` (select) + `note` (textarea) par item. En-tête : date EDL, contradictoire (checkbox), date de signature (conditionnelle).

**Dominant element:** La checklist mobilier 12 items (Miller's Law — groupés par décret, pas alphabétique).

### LOC-03 — Affichage EDL (`/baux/:id/edl/entree` et `/baux/:id/edl/sortie`)

**States:**
1. **EDL inexistant** — empty-state avec CTA "Enregistrer l'EDL d'entrée" / "Enregistrer l'EDL de sortie".

4. **Warning EDL d'entrée absent** (EDL sortie uniquement) — `banniere-warning` non-bloquante (D-85).

**Dominant element:** La liste structurée des 12 items inventaire.

**Encart EDL sur fiche Bail (`/baux/:id`) :** Section `<h2>État des lieux</h2>` ajoutée à `baux/detail.ejs`. Affiche 0/1/2 EDL avec liens vers chaque page. Warning si EDL d'entrée absent et bail actif.

## Interaction Patterns

### Forms (D-49 Phase 1)

- 1 colonne, label au-dessus de chaque champ.
- Validation inline **au blur** — jamais au submit uniquement.
- Required fields: attribut `required` + `<span aria-hidden="true">*</span>` dans le label.
- Erreur inline: `<span id="{id}-error" role="alert" class="error-msg">` + `aria-describedby` sur l'input.
- Submit disabled tant que les required non remplis (attribut `disabled` retiré par JS minimal inline — sans framework).
- Champs préservés sur échec de soumission (valeurs re-injectées via EJS).

**Champs spécifiques Phase 3:**

| Champ | Type HTML | Validation |
|-------|-----------|------------|
| `date_emission` Diagnostic | `<input type="date">` | Required. Au blur: format ISO valide. |
| `type` Diagnostic | `<select>` avec 4 options | Required. Options: DPE / Gaz / Électricité / ERP. |
| `irl_trimestre` | `<input type="text" pattern="[0-9]T[0-9]{4}">` | Required. Format "1T2026". Hint: "ex. 1T2026 (1er trimestre 2026)". |
| `irl_valeur` | `<input type="number" step="0.01" min="1">` | Required. Au blur: nombre positif. |
| `date_edl` | `<input type="date">` | Required. Warning non-bloquant si sortie < debut+duree (D-84). |
| `contradictoire` | `<input type="checkbox">` | Non required — faux par défaut. |
| `date_signature` | `<input type="date">` | Conditionnellement visible si `contradictoire` coché. |
| Items mobilier | `<input type="checkbox">` × 12 dans `<fieldset>` | Non required individuellement. Warning si obligatoires absents (D-98). |
| `etat` item EDL | `<select>` : bon / moyen / dégradé | Required si `present: true`. |
| `note` item EDL | `<textarea rows="2">` | Optionnel. |

### Tables (D-41 Phase 1)

Réutilisation du partial `data-table.ejs` pour:
- Table des diagnostics (fiche Bien) : colonnes Type / Date d'émission / Date d'expiration / Statut / Actions.
- Table historique des indexations (fiche Bail) : colonnes Date d'effet / IRL avant / IRL après / Loyer avant / Loyer après / Appliquée?.
- Toutes les colonnes numériques (loyers, valeurs IRL) right-aligned (`class="numeric"`).
- Sticky header, zebra discrète, actions row-hover.

### Banners et alertes

| Partial | `role` | `aria-live` | Condition de déclenchement |
|---------|--------|-------------|---------------------------|
| `banniere-success` existant | `status` | `polite` | Post-action réussie (enregistrement EDL, indexation appliquée) |
| `banniere-warning` existant | `alert` | `assertive` | Warning non-bloquant (diagnostic expiré, EDL d'entrée absent, items mobilier manquants) |
| `warning-live` existant | `status` | `polite` | Warnings delta inventaire sur EDL sortie |
| Banner bloquant gel loyer (nouveau) | `alert` | `assertive` | DPE F/G — implémenté inline dans le template wizard IRL, pas via partial générique |

### Wizard IRL — navigation

- Indicateur d'étape: `<ol>` numérotée horizontale en haut du wizard (étapes 1→5 avec label textuel, étape courante `aria-current="step"`).
- Bouton "Retour" présent à chaque étape sauf la 1ère. `<a href="..." role="button" class="secondary">Retour</a>` (lien, pas form submit — back = safe, D-49/UX_DESIGN §Flow).
- Bouton "Annuler" présent à chaque étape. Renvoie vers `/baux/:id` sans modification d'état.
- Aucun `history.pushState` — navigation pure par liens Fastify (SSR strict D-03).

### Destructive actions (D-50 Phase 1)

Phase 3 n'introduit pas de suppression directe de nouveaux agrégats (les diagnostics sont corrected via soft-delete + nouvel enregistrement, les EDL idem D-89, les indexations sont append-only D-96). Aucun nouveau `confirm-dialog` destructif au-delà des patterns existants.

Exception: si un soft-delete d'EDL est exposé à l'UI (annulation), utiliser `confirm-dialog.ejs` existant avec `confirmLabel: 'Annuler cet EDL'`.

Helpers purs (pas d'accès session, pas d'effet de bord) injectés dans les locals EJS:

| Helper | Signature | Exemple sortie |
|--------|-----------|----------------|
| `formaterClasseDpe(classe)` | `(string | null) => string` | `"F"` → `"DPE F"`, `null` → `"Non renseigné"` |
| `formaterTypeDiagnostic(type)` | `(TypeDiagnostic) => string` | `"dpe"` → `"DPE"`, `"gaz"` → `"Gaz"`, `"elec"` → `"Électricité"`, `"erp"` → `"ERP (risques et pollutions)"` |
| `formaterEtatItem(etat)` | `('bon' | 'moyen' | 'degrade' | null) => string` | `"degrade"` → `"Dégradé"`, `null` → `"—"` |
| `formaterTrimestreIRL(trimestre)` | `(string) => string` | `"1T2026"` → `"1er trimestre 2026"` |
| `formaterStatutDiagnostic(dateExp, today)` | `(PlainDate | null, PlainDate) => string` | `null` → `"Illimitée (ERP)"`, date passée → `"Expiré le {date}"`, date future → `"Valide jusqu'au {date}"` |
| `formaterRaisonNonApplication(raison)` | `(string | null) => string` | `"gel_dpe"` → `"Gel DPE"`, `"refus_bailleur"` → `"Choix du bailleur"`, `null` → `"Appliquée"` |

Tous les helpers sont purs et sans dépendance sur `locals` au moment de l'appel (déterministes — alignés BDD_PRACTICES.md port `Clock`).
