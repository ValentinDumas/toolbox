## Design System

| Property | Value |
|----------|-------|
| Tool | none — Pico.css classless (D-20) |
| Preset | not applicable |
| Component library | none — semantic HTML styled by Pico.css |
| Icon library | none V1 — use text labels exclusively; SVG inline only if icon-only button requires `aria-label` |

**Rationale (D-20):** Pico.css chosen for sobriety, classless approach (~10 ko), and semantic HTML out-of-the-box. Tailwind deferred to Phase 7 re-evaluation. No shadcn, no Radix, no React.

## Spacing Scale

Declared values — base 8px, multiples of 4 only (D-46). Never use arbitrary pixel values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap, inline badge padding, legend gap |
| sm | 8px | Input vertical padding, inline button padding |
| md | 16px | Field group gap, card internal padding, nav item padding |
| lg | 24px | Section header margin-bottom, form section gap |
| xl | 32px | Page content top padding, major card gap |
| 2xl | 48px | Wizard step gap, page-level vertical rhythm |
| 3xl | 64px | Full-page empty state vertical centering offset |

Exceptions:
- Touch target minimum 44px height for all interactive elements (WCAG 2.5.5 AA) — achieved via `min-height: 44px` on `<button>`, `<a>`, `<input>`.
- Sidebar width: 240px fixed (not on 8-pt scale; fixed layout value, not spacing token).

## Typography

Pico.css sets sensible base defaults — override only where specified below. All sizes in px for contract clarity; implement as `rem` in CSS (1rem = 16px base).

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Default prose, form labels, table cells (D-48) |
| Small / Label | 14px | 400 (regular) | 1.4 | Helper text, error messages, table captions — never below 14px |
| Heading (h2/h3) | 20px | 600 (semibold) | 1.2 | Section headings, wizard step title, sidebar section header |
| Display (h1) | 28px | 600 (semibold) | 1.2 | Page title — one per page (D-44, WCAG 1.3.1) |

Rules:
- Max 3 font sizes per view (D-45). Display (28px) + Heading (20px) + Body (16px) in standard views; Small (14px) replaces Body only in dense table/error contexts.
- Max 2 weights: 400 and 600. No italic for UI chrome (only for legal/document quotes).
- No `text-transform: uppercase` in sentences (D-48).
- Line length: 60–80 characters per line for paragraphs and form helper text (UI_DESIGN.md).

## Color

Pico.css provides a semantic CSS variable system (`--pico-*`). Map the 60/30/10 contract onto Pico variables below.

| Role | Pico variable / Value | Usage |
|------|-----------------------|-------|
| Dominant (60%) | `--pico-background-color` — `#ffffff` light / `#11191f` dark | Page background, form backgrounds |
| Secondary (30%) | `--pico-card-background-color` — `#f5f7fa` light / `#1c2a34` dark | Sidebar, wizard step container, data cards, nav |
| Accent (10%) | `--pico-primary` — Pico default blue `#1095c1` | See reserved-for list below |
| Destructive | `--pico-del-color` mapped to `#d32f2f` (red 700) | Delete confirmation button only |
| Semantic warning | amber `#e65100` | Not used in Phase 1 (no financial alerts) |
| Semantic success | green `#2e7d32` | Not used in Phase 1 |

**Accent reserved for (explicit list — D-47):**
1. Primary submit button in each wizard step ("Créer le bien", "Créer le locataire", "Créer le bail")
2. Primary CTA in empty states ("Créer votre premier bien")
3. Active/selected nav item in sidebar (left border highlight + text color)
4. Focus ring on interactive elements (WCAG 2.4.7)

**Accent is NOT used for:** table zebra rows, card borders, helper text, breadcrumbs, read-only data fields.

Rules (D-44, D-47):
- Contrast ratio: body text on dominant bg ≥ 4.5:1; large text/headings ≥ 3:1.
- Never use color as the only differentiator — always pair with icon or text label.
- Semantic colors (red/amber/green) used exclusively for their semantic meaning.

## Interaction Contract

### Forms (D-49)

- One-column layout, always.
- Label above field — no placeholder-only labels.
- Validation trigger: **on blur** (leave field), not on submit.
- Required fields: `required` attribute + asterisk `*` in label + `<span aria-hidden="true">*</span>`.
- Submit button: `disabled` until all `required` fields are filled (client-side via JS progressive enhancement; server validates unconditionally).
- On validation error: preserve all user input; display error message directly below the field; link via `aria-describedby`; add `role="alert"` to the error region.
- On server error (POST failure): re-render form with filled values and top-level `role="alert"` banner.

### Tables (D-41)

- `<table>` with `<caption>` (screen-reader visible, can be visually hidden with `sr-only` if heading context is clear).
- `<th scope="col">` on all column headers.
- Numbers (loyer HC, surface, montant charges, dépôt de garantie): right-aligned.
- Text (nom, adresse, type): left-aligned.
- Zebra striping via CSS `:nth-child(even)` — subtle, not distracting. No row hover + zebra simultaneously.
- Sticky `<thead>` on scroll for lists with more than ~10 rows.
- Sort indicators: show current sort column with `▲` / `▼` appended to `<th>` text; `aria-sort="ascending"` / `"descending"` attribute.
- Row actions (edit, delete): visible on `:hover` and `:focus-within` — not permanently visible columns.

### Sidebar (D-40)

- Fixed left, 240px wide.
- Nav items: `<nav>` > `<ul>` > `<li>` > `<a>`.
- Active item: accent left border (4px) + accent text color.
- Items in Phase 1: "Biens", "Locataires", "Baux".
- `aria-current="page"` on active link.

### Breadcrumbs (D-40)

- `<nav aria-label="Fil d'Ariane">` > `<ol>` > `<li>` links.
- Present on every non-wizard page.
- Example: Accueil > Biens > Appartement Rue de la Paix.

### Destructive actions (D-50)

- Delete on Bien, Lot, Locataire, Bail each require a confirmation step.
- Confirmation pattern: inline confirmation within the row (show "Confirmer la suppression ?" + "Oui, supprimer" button + "Annuler" link) OR dedicated confirmation page `/biens/:id/supprimer`.
- "Supprimer définitivement" button styled with destructive color (`#d32f2f`).
- "Annuler" is a plain `<a>` link, not a styled button, to prevent misclick.
- Undo not possible in Phase 1 (no soft-delete confirmed yet — DP-01 deferred to plan-phase). Confirmation dialog is therefore mandatory.

### Feedback states (UI_DESIGN.md)

Every interactive element has 4 states: default, hover, active (`:active`), disabled. Pico.css covers these for `<button>`, `<input>`, `<a>` by default — do not override without explicit reason.

- Loading: form submit button text changes to "Enregistrement…" + `aria-busy="true"` on the form. No spinner animation (avoids motion concerns).
- Success: redirect to list page with query param `?success=created` triggering a top-of-page `<p role="status" aria-live="polite">` banner that auto-dismisses after 4 s.
- Error: see form validation above.

## Accessibility Contract (WCAG 2.1 AA — D-44)

| Rule | Requirement |
|------|-------------|
| Contrast | Body text on bg ≥ 4.5:1; headings ≥ 3:1; UI components ≥ 3:1 |
| Keyboard | Every action reachable via Tab; logical tab order; no keyboard trap (except modal confirmation) |
| Focus | Visible focus indicator always — Pico default; never `outline: none` without replacement |
| Heading structure | One `<h1>` per page; headings not skipped (h1 → h2 → h3) |
| Labels | `<label for="...">` on every `<input>`, `<select>`, `<textarea>` |
| Errors | `aria-describedby` links input to error message; `role="alert"` on dynamic error regions |
| Tables | `<th scope="col">` on all column headers; `<caption>` or `aria-label` on each table |
| Modals | Focus trapped inside; `Escape` closes; focus returns to trigger element on close |
| Motion | `prefers-reduced-motion: reduce` — disable progress indicator animation, disable success banner fade |
| Language | `<html lang="fr">` |
| Landmarks | `<main>`, `<nav aria-label="Menu principal">`, `<nav aria-label="Fil d'Ariane">`, `<footer>` |
