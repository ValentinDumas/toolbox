# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Activation — Bien, Locataire, Bail.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | D-20: Pico.css classless — no component framework |
| Preset | not applicable | Greenfield, no components.json |
| Component library | none — semantic HTML + Pico.css | D-20 |
| Icon library | none V1 — use Unicode symbols or inline SVG aria-hidden | default |
| Font | system-ui (Pico.css default stack) | D-20 default |

**Rationale:** Pico.css is classless: semantic HTML elements are styled out-of-the-box. EJS partials serve as the component model (D-11). No Tailwind, no shadcn, no Radix.

## Spacing Scale

All values are multiples of 4. No arbitrary pixel values. Source: D-46.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, tight inline padding |
| sm | 8px | Input padding, compact element spacing, table cell padding |
| md | 16px | Default form field gap, card padding, nav item spacing |
| lg | 24px | Section padding, fieldset gap |
| xl | 32px | Layout column gap, major form section break |
| 2xl | 48px | Page vertical padding top/bottom |
| 3xl | 64px | Wizard step container max vertical offset |

Exceptions: Touch/click targets minimum 44px height (WCAG 2.5.5 AAA — applied as best practice). This is achieved via `padding` not a spacing token override.

## Typography

System-ui stack, no external font load (local-first: works offline). Source: D-48.

| Role | Size | Weight | Line Height | Max Line Length |
|------|------|--------|-------------|-----------------|
| Body | 16px | 400 (regular) | 1.5 | 72 chars |
| Label | 14px | 600 (semibold) | 1.4 | — |
| Heading | 20px | 600 (semibold) | 1.2 | 60 chars |
| Display | 28px | 600 (semibold) | 1.2 | 50 chars |

Rules:
- Maximum 3 distinct sizes per view (heading + body + label — display reserved for wizard step title only).
- Maximum 2 weights across any single view: 400 regular + 600 semibold.
- Body text never below 16px. No all-caps in sentences (decorative only).
- Numbers in tables: 16px, 400, right-aligned.

## Color

Pico.css default palette applied with semantic overrides. All colors meet WCAG 2.1 AA (4.5:1 body, 3:1 large text). Color is never the sole differentiator — always paired with icon or text label. Source: D-44, D-47.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (light) / `#13171f` (dark — Pico default) | Page background, surface |
| Secondary (30%) | `#f3f4f6` (light) / `#1e2430` (dark) | Sidebar nav, cards, table zebra row |
| Accent (10%) | `#1d4ed8` (Pico primary blue) | See reserved list below |
| Destructive | `#dc2626` (Pico red-600 equivalent) | Destructive actions only — delete confirm button |
| Warning | `#d97706` (Pico amber) | Non-blocking alerts (future phases); not used Phase 1 |
| Success | `#16a34a` (Pico green) | Persisted confirmation banner |

Accent reserved for (exhaustive — not "all interactive elements"):
1. Primary submit button ("Enregistrer le bien", "Enregistrer le locataire", "Enregistrer le bail") — filled background.
2. Wizard step indicator — active step circle fill.
3. Focused input border ring (complement to browser default).
4. "Ajouter un lot" inline add-row link within Bien form.
5. Active sidebar navigation item — left border indicator.

Everything else (secondary actions, table row hover, form fields at rest) uses secondary surface or default Pico styling. The accent color does not appear on informational text, headings, or decorative elements.

## Component Patterns

EJS partials serve as the component model. Each partial receives typed `locals`. No JS framework.

### Layout Shell (`partials/layout.ejs`)

```
+-----------------------------------------+
| <header> App name + nav trigger          |
+---------+-------------------------------+
| <nav>   |  <main>                       |
| Sidebar |   <nav aria-label="breadcrumb">|
| - Biens |   <h1> Page title             |
| - Locat.|   [content]                   |
| - Baux  |                               |
+---------+-------------------------------+
```

- Sidebar: `<nav aria-label="Navigation principale">`, `<ul>` of `<li><a>` items.
- Active item: left border accent `#1d4ed8` + bold label (semibold 600).
- No JS for sidebar toggle V1 (always visible, no mobile breakpoint V1).

### Wizard Shell (`partials/wizard-layout.ejs`)

```
+-----------------------------------------------+
| Step indicator: [1: Bien] [2: Locataire] [3: Bail]|
| Active step circle filled accent, others muted  |
+-----------------------------------------------+
| <h1> Display 28px semibold — step title         |
| <form> Single-column content                    |
| [Back link]          [Primary CTA button]       |
+-----------------------------------------------+
```

- Step indicator: `<ol aria-label="Étapes">` with `<li aria-current="step">` on active.
- Back link: `<a href="...">← Étape précédente</a>` — plain link, no button styling.
- Primary CTA: full-width at small viewports, right-aligned at ≥640px.

### Form Partial (`partials/form-field.ejs`)

```
<div class="field">
  <label for="[id]">[Label] <span aria-hidden="true">*</span></label>
  <input id="[id]" name="[name]" required aria-describedby="[id]-error">
  <span id="[id]-error" role="alert" class="error-msg" hidden>[error copy]</span>
</div>
```

- Label above input — never placeholder as label.
- Error revealed on blur via inline `<script>` or server-side re-render (POST → redirect on success, re-render with errors on failure).
- Required fields: `required` HTML attribute + `*` visual marker. No "optional" labels.
- Submit disabled until all required fields filled (`required` + browser native validation; JS progressive enhancement only).

### Data Table Partial (`partials/data-table.ejs`)

```html
<table aria-label="[description]">
  <caption class="sr-only">[description]</caption>
  <thead>
    <tr>
      <th scope="col">[Label]</th>
      <th scope="col" class="numeric">[Label]</th>  <!-- right-align -->
      <th scope="col" class="sr-only">Actions</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>[text — left-align]</td>
      <td class="numeric">[number — right-align]</td>
      <td class="row-actions">
        <a href="/biens/:id/modifier">Modifier</a>
        <button ...>Supprimer</button>
      </td>
    </tr>
  </tbody>
</table>
```

- Zebra: `tbody tr:nth-child(odd)` background secondary surface. No row hover + zebra together.
- Sticky header: `thead th { position: sticky; top: 0; }`.
- Sort indicators: `aria-sort="ascending|descending|none"` on `<th>` + visible Unicode caret (↑ ↓).
- Actions visible on row hover (CSS `:hover` on `<tr>` reveals `.row-actions` links). No permanent action columns.
- Numbers right-aligned (`text-align: right`), text left-aligned.

## Interaction & State Contracts

### Form Validation Flow

1. User focuses a required field → no validation yet.
2. User blurs a required field empty → error message revealed inline (not page-level).
3. User corrects field → error hidden immediately on valid input (blur or input event).
4. Submit button enabled state: native HTML `required` + browser constraint validation.
5. On POST failure (server error) → form re-rendered with pre-filled values + server-side error messages. Input never cleared.
6. On POST success → redirect to list or detail page + success banner (`aria-live="polite"`).

### Success Banner

Persisted message displayed once after successful save/redirect:

```
"[Entity] enregistré avec succès."  — rendered in accent-success green, auto-dismissed after 5 s
```

Uses `aria-live="polite"` region. No JS animation — show/hide via hidden attribute.

### Confirmation Modal (Destructive)

- Rendered server-side as `<dialog>` element (no JS framework).
- Focus trapped inside `<dialog>` (native `<dialog>` behavior in modern browsers).
- Default focus: "Annuler" button.
- `Escape` key closes dialog (native).
- Background scroll locked while open.

### Wizard State

- State persisted across steps via HTTP session cookie (server-side, `@fastify/session` or equivalent — cf. DP-05).
- On browser Back from Step 2 → Step 1 pre-filled with saved values (no data loss).
- Wizard shown only on first launch (cf. DP-06 detection strategy).
- On completing Step 3 → redirect to `/biens` list with success banner showing all 3 entities created.

## Accessibility Contract

All items are mandatory — verified by UI-SPEC checker dimension and CI gate.

| Check | Requirement |
|-------|-------------|
| Contrast body text | 4.5:1 minimum against background |
| Contrast large text (≥18px or ≥14px bold) | 3:1 minimum |
| Contrast UI components (buttons, inputs, icons) | 3:1 against adjacent color |
| Keyboard navigation | All interactive elements reachable via Tab, logical order |
| Focus indicator | Visible on every focusable element — never `outline: none` without replacement |
| Modal focus trap | `<dialog>` traps focus; restores on close |
| Escape closes modal | Required for `<dialog>` |
| One `<h1>` per page | Enforced — wizard step title OR list page title |
| Heading hierarchy | Never skip levels (`h1` → `h2` → `h3`) |
| `<label>` association | Every `<input>`, `<select>`, `<textarea>` has explicit `<label for="">` |
| Error ARIA | `aria-describedby` linking input to error message |
| `role="alert"` on errors | Dynamic errors use `role="alert"` |
| Table semantics | `<th scope="col">` + `<caption>` or `aria-label` |
| Icon-only buttons | `aria-label` on every icon-only action |
| Color independence | No information conveyed by color alone (icon or text always paired) |
| `prefers-reduced-motion` | No CSS animations unless `prefers-reduced-motion: no-preference` |
| Semantic HTML | `<button>` for actions, `<a>` for navigation |
| `<fieldset>` + `<legend>` | Used for grouped inputs (Adresse group, Lot type group) |
